# 思考时长功能测试指南

## 功能说明
本次更新实现了 AI 思考时长的实时计算、前端动态显示和后端持久化保存功能。

## 测试步骤

### 1. 启动前后端服务

**后端启动：**
```bash
cd backend
python -m uvicorn app.main:app --reload --port 8000
```

**前端启动：**
```bash
cd frontend
npm run dev
```

### 2. 基础功能测试

#### 测试用例 1：思考时长实时显示
1. 打开浏览器访问前端页面（通常是 http://localhost:5173）
2. 选择一个支持"深度思考"的模型
3. 在设置中启用"深度思考"选项
4. 发送一个问题，例如："请详细解释量子纠缠的原理"
5. **观察点：**
   - 思考框展开后，标题旁应显示 "已思考 X.Xs"
   - 数字应该实时更新（每 100ms 刷新一次）
   - 思考过程中文字显示为 "思考中..."
   - 思考结束后文字变为 "已深度思考"，时长固定为最终值

#### 测试用例 2：思考时长格式
1. 发送一个需要长时间思考的复杂问题
2. **观察点：**
   - 思考时间 < 60 秒时，显示格式应为 "X.Xs"（如 "3.5s"）
   - 思考时间 ≥ 60 秒时，显示格式应为 "X 分 Y 秒"（如 "1 分 30.5 秒"）

#### 测试用例 3：历史记录回显
1. 完成一次对话后，刷新页面
2. **观察点：**
   - 历史消息的思考框应正确显示之前的思考时长
   - 显示格式应为 "思考耗时 X.Xs"（非"已思考"）
   - 时长数值应与之前看到的最终值一致

#### 测试用例 4：多版本切换
1. 对同一个问题点击"重新生成"按钮，生成多个回答版本
2. 使用版本切换按钮（← →）切换不同版本
3. **观察点：**
   - 每个版本应独立显示自己的思考时长
   - 切换版本时光长不应混淆

### 3. 兼容性测试

#### 测试用例 5：无思考模式
1. 关闭"深度思考"选项
2. 发送问题
3. **观察点：**
   - 不应显示思考框
   - 正常显示回答内容
   - 不出现任何错误

#### 测试用例 6：工具调用
1. 启用需要工具调用的功能（如网络搜索）
2. 发送问题
3. **观察点：**
   - 工具调用框正常显示
   - 思考时长显示不受影响
   - 两者状态独立，互不干扰

#### 测试用例 7：Markdown 渲染
1. 发送一个包含代码块的复杂问题
2. **观察点：**
   - Markdown 格式正常渲染
   - 代码高亮正常
   - 思考时长显示不影响内容区域

### 4. 数据持久化验证

#### 测试用例 8：数据库存储
1. 完成一次带思考的对话
2. 检查数据库（如果可访问）：
```sql
SELECT id, turns_id, meta_data 
FROM message_content 
WHERE meta_data->>'thinking_duration_ms' IS NOT NULL;
```
3. **观察点：**
   - `meta_data` JSON 字段中应包含 `thinking_duration_ms`
   - 数值应为正整数（毫秒）

### 5. 性能测试

#### 测试用例 9：内存泄漏检测
1. 连续进行 10 次以上的对话
2. 打开浏览器开发者工具，观察内存使用
3. **观察点：**
   - 内存使用应保持稳定
   - 不应持续增长（计时器应正确清理）
   - 控制台无错误信息

## 常见问题排查

### 问题 1：不显示思考时长
**可能原因：**
- 模型不支持 thinking 功能
- 前端未启用"深度思考"选项
- 后端未正确返回 think 事件

**解决方法：**
1. 检查模型配置是否包含 "thinking" feature
2. 确认会话设置中 `thinking_enabled` 为 true
3. 查看浏览器 Network 面板，确认 SSE 流中有 `{"type":"think"}` 事件

### 问题 2：时长不更新
**可能原因：**
- 计时器未正确启动
- JavaScript 执行错误

**解决方法：**
1. 打开浏览器控制台，检查是否有错误信息
2. 在 `ChatPanel.vue` 的 think 事件处理处添加断点
3. 确认 `setInterval` 被正确调用

### 问题 3：刷新后时长丢失
**可能原因：**
- 后端未正确保存 `thinking_duration_ms`
- 前端未从 `meta_data` 中读取时长

**解决方法：**
1. 检查数据库中 `meta_data` 字段是否包含时长
2. 在 `MessageItem.vue` 中添加日志，确认 `turn.thinking_duration_ms` 有值
3. 如需从 meta_data 回填，可在组件中添加：
```javascript
// 如果 turn.thinking_duration_ms 为空，从 meta_data 读取
const duration = turn.thinking_duration_ms || turn.meta_data?.thinking_duration_ms;
```
