# 知识库文件夹上传功能 - 快速使用指南

## 功能概述

现在您可以直接拖拽整个文件夹到知识库,系统会**自动保留原始的目录层级结构**。

---

## 使用方法

### 方式1: 拖拽上传(推荐) ⭐

1. 打开知识库页面
2. 从文件管理器中**直接拖拽文件夹**到上传区域
3. 系统会自动识别文件夹结构并开始上传

```
示例: 拖拽 "桌面/A项目" 文件夹
├─ A项目/
│  ├─ docs/
│  │  └─ readme.md          → 相对路径: "A项目/docs/readme.md"
│  └─ src/
│     └─ main.py            → 相对路径: "A项目/src/main.py"
```

### 方式2: 点击选择

1. 点击上传区域的"点击选择文件夹"文字
2. 在弹出的对话框中选择目标文件夹
3. 确认后开始上传

---

## 关键特性

✅ **保留目录结构**: 上传后可以在数据库中看到完整的文件夹树  
✅ **智能去重**: 同名文件夹不会重复创建  
✅ **并发控制**: 最多同时上传3个文件,避免服务器过载  
✅ **进度反馈**: 实时显示每个文件的上传和处理状态  
✅ **断点续传**: 刷新页面后未完成的任务会继续处理  

---

## 注意事项

### ⚠️ 文件大小限制
- 单个文件最大 **50MB**
- 超过限制的文件会自动跳过并提示

### ⚠️ 浏览器兼容性
- ✅ Chrome / Edge: 完全支持拖拽和点击两种方式
- ⚠️ Firefox: 可能需要用户授权访问文件系统
- ❌ Safari: 仅支持点击选择方式,不支持拖拽

### ⚠️ 空文件夹
浏览器无法上传空文件夹,如果需要保留空文件夹结构,请手动在其中添加一个占位文件(如 `.gitkeep`)。

### ⚠️ 特殊字符
文件名中包含以下字符可能导致问题:
- `/` (斜杠)
- `\` (反斜杠)
- `:` (冒号,Windows)
- `*` `?` `"` `<` `>` `|` (Windows保留字符)

建议上传前重命名这些文件。

## 常见问题

### Q: 如何查看上传后的目录结构?

A: 目前数据库已存储了完整的树形结构,但前端尚未实现树形视图组件。您可以通过以下方式验证:

```sql
-- 查询某个知识库的所有文件和文件夹
SELECT 
  id,
  display_name,
  relative_path,
  is_directory,
  parent_folder_id
FROM kb_file
WHERE knowledge_base_id = 'your_kb_id'
ORDER BY relative_path;
```

### Q: 上传过程中可以关闭页面吗?

A: 可以。后端会在后台继续处理已接收的文件。刷新页面后,正在进行中的任务会自动恢复进度显示。

### Q: 如何删除整个文件夹?

A: 当前版本需要逐个删除文件。未来会提供文件夹级别的批量删除功能。

### Q: 能否移动文件到其他文件夹?

A: 当前不支持。如需调整文件位置,请删除后重新上传到目标文件夹。

## 技术细节(开发者参考)

### 数据模型

```prisma
model KBFile {
  relativePath       String?    // 相对路径,如 "docs/api/readme.md"
  parentFolderId     String?    // 父文件夹ID(外键自引用)
  isDirectory        Boolean    // 是否为文件夹节点
}
```

### API端点

```typescript
POST /api/v1/knowledge-bases/:kb_id/files/upload
Body: FormData {
  file: File,
  relativePath?: string  // 可选,如 "docs/readme.md"
}
```

### 核心流程

```
前端拖拽
  ↓
FileSystem API 递归遍历
  ↓
提取 relativePath (如 "A/B/c.txt")
  ↓
POST 上传(携带 relativePath)
  ↓
后端解析路径
  ↓
ensureFolderStructure("A/B")
  ├─ 创建/查找 "A" (parent=null)
  └─ 创建/查找 "B" (parent=A.id)
  ↓
创建文件记录 (parent=B.id)
  ↓
后台向量化处理
```
