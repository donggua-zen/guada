# 备忘录功能实现说明

## 概述

在现有长期记忆系统的基础上，新增了"备忘录"功能，允许 AI 助手创建、读取、更新和删除结构化的笔记。

## 功能特性

### 1. 创建/更新备忘录 (`memory__memo__create_or_update`)

**用途**: 保存临时或重要的笔记、待办事项、参考资料等

**参数**:
- `title` (必需): 备忘录标题，最大64字符，作为唯一标识
- `content` (必需): 备忘录内容
- `write_mode` (可选): 写入模式
  - `overwrite` (默认): 完全替换已有内容
  - `append`: 在原有内容后追加新内容

**行为**:
- 如果标题不存在，创建新备忘录
- 如果标题已存在，根据 `write_mode` 决定覆盖或追加

**示例**:
```typescript
// 创建新备忘录
{
  title: "会议要点",
  content: "1. 讨论项目进度\n2. 确定下一步计划",
  write_mode: "overwrite"
}

// 追加到现有备忘录
{
  title: "会议要点",
  content: "3. 分配任务",
  write_mode: "append"
}
```

### 2. 删除备忘录 (`memory__memo__delete`)

**用途**: 删除不再需要的备忘录

**参数**:
- `title` (必需): 要删除的备忘录标题

**示例**:
```typescript
{
  title: "过时的笔记"
}
```

### 3. 读取备忘录 (`memory__memo__read`)

**用途**: 读取指定标题的备忘录完整内容

**参数**:
- `title` (必需): 要阅读的备忘录标题

**示例**:
```typescript
{
  title: "会议要点"
}
```

### 4. 备忘录目录注入

在每次对话的系统提示词中，会自动注入当前会话的所有备忘录标题列表，格式如下：

```
【备忘录目录】
以下是你可访问的备忘录列表，可通过标题读取具体内容：
<memo-list>
1. 会议要点
2. 用户偏好
3. 项目计划
</memo-list>
```

这使得 AI 可以知道有哪些备忘录可用，并可以根据需要读取具体内容。

## 技术实现

### 数据存储

备忘录使用现有的 `Memory` 模型存储，通过以下字段区分：

- `category`: `"memo"` (区别于长期记忆的 `"long_term"`)
- `tags`: 存储备忘录标题（作为唯一标识）
- `content`: 存储备忘录内容
- `memoryType`: `"factual"` (固定值)
- `sessionId`: 关联到具体会话

### 数据库查询优化

由于 `tags` 字段用于存储标题并进行查询，建议在 `tags` 字段上添加索引以提高查询性能。可以在 Prisma schema 中添加：

```prisma
model Memory {
  // ... 其他字段
  tags       String?
  
  @@index([tags])
  // ... 其他索引
}
```

### 工具注册

在 `MemoryToolProvider` 中注册了三个新工具：

1. `memory__memo__create_or_update`
2. `memory__memo__delete`
3. `memory__memo__read`

这些工具会自动在对话中被 AI 调用。

## 使用场景

### 场景1: 记录用户偏好

当用户提供个人偏好时，AI 可以创建备忘录：

```
用户: 我喜欢蓝色主题，并且希望使用简洁的界面
AI: [调用 memory__memo__create_or_update]
    title: "用户界面偏好"
    content: "- 主题颜色：蓝色\n- 界面风格：简洁"
    write_mode: "overwrite"
```

### 场景2: 保存对话要点

在长对话中，AI 可以总结关键点并保存：

```
AI: [调用 memory__memo__create_or_update]
    title: "项目需求总结"
    content: "1. 需要支持多语言\n2. 需要离线功能\n3. 需要数据同步"
    write_mode: "overwrite"
```

### 场景3: 后续查阅

当用户询问之前讨论的内容时：

```
用户: 我们之前讨论的项目需求是什么？
AI: [查看备忘录目录，发现"项目需求总结"]
    [调用 memory__memo__read]
    title: "项目需求总结"
    [返回内容并回答用户]
```

### 场景4: 更新过时信息

```
用户: 项目需求有变化，不需要离线功能了
AI: [调用 memory__memo__read 读取当前内容]
    [修改内容后调用 memory__memo__create_or_update]
    title: "项目需求总结"
    content: "1. 需要支持多语言\n2. ~~需要离线功能~~\n3. 需要数据同步"
    write_mode: "overwrite"
```

## 与长期记忆的区别

| 特性 | 长期记忆 | 备忘录 |
|------|----------|--------|
| 用途 | 核心事实、人格定义 | 临时笔记、待办事项、参考资料 |
| 数量限制 | 每种类型一条 | 无限制，可创建多条 |
| 标识方式 | 按类型 (factual/soul) | 按标题 (唯一) |
| 管理方式 | 自动注入到提示词 | 仅注入目录，按需读取 |
| 适用场景 | 持久化重要信息 | 灵活的信息组织 |

## 最佳实践

1. **标题简洁明确**: 标题应能清晰反映内容主题，便于查找
2. **内容结构化**: 使用列表、分段等格式组织内容
3. **及时更新**: 当信息变化时及时更新或删除过时备忘录
4. **合理使用写入模式**: 
   - 需要保留历史内容时使用 `append`
   - 需要完全替换时使用 `overwrite`
5. **避免重复**: 相似内容应合并到同一个备忘录中

## 未来扩展

可能的改进方向：

1. **备忘录分类**: 添加标签或分类系统
2. **搜索功能**: 支持按内容搜索备忘录
3. **过期时间**: 支持设置备忘录的有效期
4. **优先级**: 为备忘录添加优先级标记
5. **跨会话共享**: 允许在不同会话间共享备忘录

## 注意事项

1. 标题长度限制为64字符，超出会被拒绝
2. 标题在同一会话内必须唯一
3. 备忘录与会话绑定，删除会话会级联删除相关备忘录
4. 备忘录目录会在每次对话时注入，大量备忘录可能增加 token 消耗
