# MCP 服务器配置导入功能说明

## 📋 功能概述

MCP 服务器配置导入功能允许您快速批量导入服务器配置，无需手动逐个添加。支持标准 JSON 格式配置文件的导入。

---

## 🎯 支持的格式

### 1. 标准格式（推荐）

```json
{
  "mcpServers": {
    "ServerKey1": {
      "type": "streamableHttp",
      "description": "服务器描述",
      "isActive": true,
      "name": "服务器名称",
      "baseUrl": "https://example.com/mcp",
      "headers": {
        "Authorization": "Bearer your-token"
      }
    },
    "ServerKey2": {
      // ... 更多服务器
    }
  }
}
```

**特点：**
- ✅ 支持批量导入多个服务器
- ✅ 使用 `mcpServers` 对象包裹所有配置
- ✅ 每个服务器有唯一的键名（如 `WebSearch`、`ImageGeneration`）
- ✅ 可包含完整的配置信息（类型、描述、状态、请求头等）

### 2. 单个服务器对象格式

```json
{
  "type": "streamableHttp",
  "description": "服务器描述",
  "isActive": true,
  "name": "服务器名称",
  "baseUrl": "https://example.com/mcp",
  "headers": {
    "Authorization": "Bearer your-token"
  }
}
```

**特点：**
- ✅ 适用于导入单个服务器
- ✅ 直接提供服务器对象，无需外层包装

---

## 🔧 字段映射说明

导入的配置会自动映射到系统字段：

| 源字段 | 目标字段 | 说明 |
|--------|---------|------|
| `name` | `name` | **必填** - 服务器名称 |
| `baseUrl` 或 `url` | `url` | **必填** - 服务地址 |
| `description` | `description` | 可选 - 描述信息 |
| `headers` | `headers` | 可选 - HTTP 请求头（JSON 对象） |
| `isActive` | `enabled` | 可选 - 启用状态（默认 true） |
| `type` | - | 仅用于参考，不存储 |

## 📖 使用步骤

### 步骤 1：打开导入对话框

1. 进入设置页面 → MCP 服务器管理
2. 点击顶部的 **"导入配置"** 按钮

### 步骤 2：准备 JSON 数据

复制您的 MCP 服务器配置 JSON 数据。例如：

```json
{
  "mcpServers": {
    "WebSearch": {
      "type": "streamableHttp",
      "description": "阿里云百炼联网搜索服务",
      "isActive": true,
      "name": "阿里云百炼_联网搜索",
      "baseUrl": "https://dashscope.aliyuncs.com/api/v1/mcps/WebSearch/mcp",
      "headers": {
        "Authorization": "Bearer sk-b4ed412869c342b5b50a0f3177af5d5c"
      }
    }
  }
}
```

### 步骤 3：粘贴并导入

1. 将 JSON 数据粘贴到文本框中
2. 点击 **"导入"** 按钮
3. 系统会自动解析并批量创建服务器

### 步骤 4：查看结果

导入完成后会显示：
- ✅ 成功导入的服务器数量
- ❌ 失败的服务器数量及原因（如有）

## ⚠️ 注意事项

### 1. 必填字段验证

导入的每个服务器必须包含：
- ✅ `name` - 服务器名称
- ✅ `baseUrl` 或 `url` - 服务地址

缺少任一字段会导致该服务器导入失败。

### 2. 数据转换规则

- `isActive` → `enabled`（布尔值直接转换）
- `baseUrl` → `url`（统一使用 url 字段存储）
- `headers` 保持为 JSON 对象格式
- 缺失的字段会使用默认值：
  - `description`: 自动生成（"导入自配置文件：xxx"）
  - `enabled`: `true`
  - `headers`: `{}`

### 3. 错误处理

以下情况会导致导入失败：

```javascript
// ❌ 无效的 JSON 格式
{ name: "test", url: "http://..." }  // 缺少引号

// ❌ 缺少必填字段
{ "name": "test" }  // 缺少 baseUrl/url

// ❌ 无法识别的格式
{ "servers": [...] }  // 不是 mcpServers 且不是有效对象
```

### 4. 批量导入行为

- ✅ 即使部分服务器导入失败，成功的仍会被保存
- ✅ 会显示详细的错误报告，指出哪些服务器失败及原因
- ✅ 导入成功后会自动刷新列表

## 🔍 常见问题

### Q1: 导入后看不到服务器？

**A:** 
1. 检查是否显示成功提示
2. 刷新页面查看列表
3. 确认服务器是否为"已禁用"状态（灰色标签）

### Q2: 提示"JSON 格式错误"？

**A:**
1. 确保使用双引号 `"` 而非单引号 `'`
2. 检查逗号是否正确分隔每个字段
3. 确保括号正确匹配
4. 可以使用在线 JSON 格式化工具验证

### Q3: 部分服务器导入失败？

**A:**
1. 查看错误提示信息
2. 常见原因：
   - 缺少必填字段（name 或 baseUrl）
   - URL 格式不正确
   - 服务器名称重复
3. 修正后重新导入失败的服务器

### Q4: 可以覆盖现有的服务器吗？

**A:** 
当前版本不会覆盖同名服务器。如果需要更新现有服务器：
1. 先删除旧服务器
2. 重新导入新配置
或者
1. 手动编辑现有服务器配置

## 🚀 小贴士

### 快速开始

1. 复制 [`导入配置示例.json`](./导入配置示例.json) 的内容
2. 在 MCP 服务器管理页面点击"导入配置"
3. 粘贴并点击导入
4. 立即拥有 3 个预配置的服务器！

### 导出配置

虽然当前不支持一键导出，但您可以：
1. 在浏览器控制台执行：
   ```javascript
   const config = { mcpServers: {} }
   servers.forEach(s => {
     config.mcpServers[s.name] = {
       type: 'streamableHttp',
       description: s.description,
       isActive: s.enabled,
       name: s.name,
       baseUrl: s.url,
       headers: s.headers
     }
   })
   console.log(JSON.stringify(config, null, 2))
   ```
2. 复制输出的 JSON 即可保存配置

### 批量管理

当需要部署到多个环境时：
1. 在开发环境配置好所有服务器
2. 使用上述方法导出配置
3. 在生产环境一键导入
4. 仅需修改 API Key 等敏感信息
