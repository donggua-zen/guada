## 🔑 关键实现细节

### 1. HTTP 请求头格式转换

#### 多行文本 → 对象
```javascript
const parseHeaders = (text) => {
    if (!text || typeof text !== 'string') {
        return {}
    }
    
    const headers = {}
    const lines = text.split('\n')
    
    for (const line of lines) {
        const trimmedLine = line.trim()
        if (!trimmedLine) continue
        
        const colonIndex = trimmedLine.indexOf(':')
        if (colonIndex === -1) continue
        
        const key = trimmedLine.substring(0, colonIndex).trim()
        const value = trimmedLine.substring(colonIndex + 1).trim()
        
        if (key && value) {
            headers[key] = value
        }
    }
    
    return headers
}
```

#### 对象 → 多行文本
```javascript
const headersText = server.headers ? Object.entries(server.headers)
    .map(([key, value]) => `${key}: ${value}`)
    .join('\n') : ''
```

### 2. 错误处理

所有 API 调用都包含完整的错误处理：

```javascript
try {
    const response = await apiService.fetchMcpServers()
    // 处理响应数据
} catch (error) {
    console.error('加载 MCP 服务器失败:', error)
    toast.error(error.message || '加载失败')
    servers.value = []
}
```

### 3. 数据验证

表单验证规则：
- 服务器名称：必填，2-50 字符
- 服务地址：必填，URL 格式验证
- 描述信息：可选
- HTTP 请求头：可选，自动解析
- 启用状态：布尔值

## ⚠️ 注意事项

### 1. CORS 配置

确保后端已正确配置 CORS，允许前端域名访问：

```python
# backend/app/__init__.py
app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.ALLOWED_ORIGINS,  # 包含前端地址
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 2. Token 认证

所有 API 请求都会自动携带 JWT Token：
- Token 存储在 localStorage 中
- 通过 Axios 拦截器自动添加到请求头
- 未登录用户会被重定向到登录页

### 3. 数据格式

后端返回的数据格式必须一致：
```javascript
{
  code: 200,          // 状态码
  message: 'success', // 消息
  data: {}            // 实际数据
}
```

### 4. 错误处理

统一的错误处理模式：
```javascript
catch (error) {
    console.error('操作失败:', error)
    toast.error(error.message || '操作失败')
}
```
