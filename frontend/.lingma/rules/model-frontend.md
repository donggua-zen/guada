## Vue 项目前端开发规范

### 图标规范
在前端 Vue 项目中使用图标时，必须遵循以下步骤：

1. **搜索图标库**：首先检查项目中安装的图标库（如 `@iconify/vue`、`element-plus/icons-vue` 或自定义的 `icons` 目录）。
2. **确认图标存在**：在对应的图标库文档或源代码中搜索所需的图标名称，确保该图标已被导出。
3. **正确导入**：仅导入已确认存在的图标，避免使用未定义的图标导致运行时错误。

**示例检查流程**：
- 若使用 Element Plus：查阅 [Element Plus Icons](https://element-plus.org/zh-CN/component/icon.html) 官方列表。
- 若使用本地 SVG 集合：检查 `src/assets/icons/` 目录下的文件列表或自动生成的类型定义文件。

### CSS规范

4. **样式编写规范**：在设计组件样式时，应优先使用 **Tailwind CSS** 工具类。
   - 避免在 `<style>` 标签中编写自定义 CSS，除非遇到 Tailwind 无法覆盖的复杂场景。
   - 利用原子类快速构建布局、间距、颜色及响应式效果，保持代码简洁且一致。
   - 若需复用样式组合，建议使用 `@apply` 指令或在组件中提取为独立的逻辑单元，而非创建全局类名。
