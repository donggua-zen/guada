### 优化 2: 自适应长度归一化 ⭐⭐⭐

#### 问题

```python
# 优化前：硬编码的平均长度
avg_length = 500  # ⚠️ 假设值
calibrated = abs(raw_score) * 1000000  # ⚠️ 放大倍数太大
length_factor = (avg_length / content_length) ** 0.5
# 结果：分数异常高（百万级别）
```

**观察到的现象**:
```
Top-1: backend/knowledge_base\HYBRID_SEARCH_... | 3453910.0665 | 相关性: 0 ❌
Top-2: backend-ts/BM25_FIX_SUMMARY.md           | 2987376.7197 | 相关性: 0 ❌
```

#### 解决方案

```python
def _calibrate_bm25_score(self, raw_score: float, content_length: int, avg_length: float = None) -> float:
    """改进的 BM25 分数校准"""
    if avg_length is None:
        avg_length = 500  # 默认值（向后兼容）
    
    # 1. 基础校准（缩小倍数，避免极端值）
    calibrated = abs(raw_score) * 100  # 从 1000000 降至 100
    
    # 2. 长度归一化（使用实际平均值）
    length_factor = (avg_length / content_length) ** 0.5
    
    # 3. 添加上下限（避免极端值）
    length_factor = max(0.3, min(3.0, length_factor))
    
    calibrated *= length_factor
    
    return calibrated

# 使用时计算实际平均长度
corpus = [tokenize(chunk['content']) for chunk in candidate_chunks]
avg_length = sum(len(tokens) for tokens in corpus) / len(corpus)
logger.info(f"语料库平均长度: {avg_length:.1f} tokens")

for i, (chunk, raw_score) in enumerate(zip(candidate_chunks, scores)):
    if raw_score > 0:
        calibrated_score = self._calibrate_bm25_score(raw_score, len(chunk['content']), avg_length)
```

#### 效果

```
语料库平均长度: 711.8 tokens（查询 1）
语料库平均长度: 730.0 tokens（查询 2）
语料库平均长度: 730.2 tokens（查询 3）
```

**优势**:
- ✅ 使用实际计算的平均长度（~720），而非硬编码的 500
- ✅ 缩小放大倍数（100 vs 1000000）
- ✅ 添加上下限（0.3-3.0），避免极端值
- ✅ 分数范围更合理（不再出现百万级别）

## 🎯 进一步优化方向

### P1: 融合 FTS5 和 rank-bm25 分数（推荐）⭐⭐⭐

**思路**: 不完全依赖 rank-bm25，而是融合两种分数

```python
# 1. FTS5 原始分数
fts5_scores = [row[-1] for row in rows]

# 2. rank-bm25 重新计算
bm25_scores = bm25.get_scores(query_tokens)

# 3. 归一化到 [0, 1]
def normalize(scores):
    min_s, max_s = min(scores), max(scores)
    if max_s == min_s:
        return [0.5] * len(scores)
    return [(s - min_s) / (max_s - min_s) for s in scores]

fts5_normalized = normalize(fts5_scores)
bm25_normalized = normalize(bm25_scores)

# 4. 加权融合
alpha = 0.3  # FTS5 权重
beta = 0.7   # rank-bm25 权重
final_scores = [alpha * f + beta * b for f, b in zip(fts5_normalized, bm25_normalized)]
```

**预期效果**: MRR 从 0.43 提升至 **0.55-0.65**

### P2: 调整候选集占比策略

**当前**: 固定 25%  
**改进**: 根据查询复杂度动态调整

```python
def get_candidate_size(total_chunks: int, top_k: int, query_complexity: str) -> int:
    """根据查询复杂度动态调整候选集占比"""
    if query_complexity == 'simple':
        ratio = 0.20  # 简单查询，少召回
    elif query_complexity == 'complex':
        ratio = 0.30  # 复杂查询，多召回
    else:
        ratio = 0.25  # 默认
    
    ideal_size = int(total_chunks * ratio)
    return max(top_k * 5, min(ideal_size, top_k * 20))
```

**预期效果**: MRR 从 0.43 提升至 **0.48-0.53**

### P3: 优化 FTS5 查询策略

**当前**: 简单的 OR 查询  
**改进**: 多级查询策略

```python
# 级别 1: 精确短语匹配
phrase_query = f'"{query}"'
candidates = execute_fts5(phrase_query, limit=top_k * 3)

# 级别 2: 术语 OR 查询（如果候选不足）
if len(candidates) < top_k * 5:
    or_query = ' OR '.join(escaped_tokens)
    new_candidates = execute_fts5(or_query, limit=top_k * 5 - len(candidates))

```
