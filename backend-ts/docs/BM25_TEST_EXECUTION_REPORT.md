#### 问题 2: 大小写不匹配导致得分为 0
**现象**: "DV430FBM" 查询得分为 0  
**原因**: 
- 查询词分词: `['dv430fbm']`（小写）
- 文档分词: `['DV430FBM', '-', 'N20']`（大写）
- 大小写不一致导致无法匹配

**修复**: 在中英混合文本分词时，将英文部分统一转为小写

```python
# Python
if has_chinese:
    tokens = self.tokenize_chinese(text)
    return [t.lower() if t.isascii() else t for t in tokens]
```

```typescript
// TypeScript
if (hasChinese) {
  const tokens = this.jieba.cut(text, true);
  return tokens.map(t => /^[a-zA-Z0-9]+$/.test(t) ? t.toLowerCase() : t);
}
```

**效果**: 
- 修复前: 得分为 0 ❌
- 修复后: 文档 3 得分 0.3565，文档 0 得分 0.2320 ✅

## 🔍 根本原因分析

### 1. 分词器行为差异

#### Python jieba vs TypeScript @node-rs/jieba

**测试案例**: `"DV430FBM-N20 冰箱"`

```python
# Python jieba
list(jieba.cut("DV430FBM-N20 冰箱"))
# 可能结果: ['DV430FBM', '-', 'N20', '冰箱'] 或 ['D', 'V', '4', '3', '0', ...]
```

```typescript
// TypeScript @node-rs/jieba
jieba.cut("DV430FBM-N20 冰箱", true)
// 可能结果: ['DV430FBM', 'N20', '冰箱'] 或 ['D', 'V', '4', ...]
```

**影响**:
- 未登录词（如产品型号）的分词结果不确定
- 导致 TF 和 DF 计算差异

### 2. BM25 实现细节差异

#### IDF 公式验证

两端使用相同公式：
```
IDF = log((N - df + 0.5) / (df + 0.5) + 1)
```

**但可能存在**:
- 浮点数精度差异（极小）
- 对数函数实现差异（math.log vs Math.log）

#### 长度归一化

```
denominator = tf + k1 * (1 - b + b * (docLength / avgDocLength))
```

**潜在差异**:
- 文档长度计算方式（字符数 vs token 数）
- 平均长度的计算精度

### 3. 分数缩放因子

**观察**: TS 端分数普遍为 Python 的 1.9-2.3 倍

**可能原因**:
1. **分词粒度不同**: TS 端可能产生更少的 token，导致 TF 更高
2. **IDF 基数差异**: 虽然公式相同，但 df 的计算可能不同
3. **库实现差异**: rank-bm25 可能有内部优化或调整
