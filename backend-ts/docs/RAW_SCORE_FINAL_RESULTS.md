## 💡 为什么原始分数效果更好？

### 原因 1: 避免双重归一化 ⭐⭐⭐

**rank-bm25 内部已经做了完善的长度归一化**:
```python
# rank-bm25 内部公式（标准 Okapi BM25）
score = IDF * (TF * (k1 + 1)) / (TF + k1 * (1 - b + b * |d|/avgdl))
#                                          ^^^^^^^^^^^^^^^^^^^^^^^^
#                                          长度归一化已经完成
```

**我们的额外校准（有问题）**:
```python
def _calibrate_bm25_score(self, raw_score: float, content_length: int, avg_length: float) -> float:
    calibrated = abs(raw_score) * 100
    
    # ❌ 额外的长度归一化（与 rank-bm25 重复）
    length_factor = (avg_length / content_length) ** 0.5
    length_factor = max(0.3, min(3.0, length_factor))
    
    calibrated *= length_factor  # ❌ 双重归一化
    
    return calibrated
```

**后果**:
- ❌ 短文档被过度加权
- ❌ 长文档被过度惩罚
- ❌ 排序偏离最优解

**移除校准后**:
- ✅ 保持 rank-bm25 的原始排序
- ✅ 避免双重归一化
- ✅ 更接近理论最优

---

### 原因 2: rank-bm25 的参数已经过优化 ⭐⭐

**rank-bm25 使用标准参数**:
```python
# 默认参数
k1 = 1.2  # TF 饱和参数
b = 0.75  # 长度归一化参数

# 这些参数经过大量实验验证，在大多数场景下表现良好
```

**我们的额外调整可能不合适**:
```python
# 我们使用的指数
length_factor = (avg_length / content_length) ** 0.5

# 这个指数可能与 rank-bm25 的 b=0.75 不匹配
# 导致归一化强度不一致
```

**移除校准后**:
- ✅ 保持 rank-bm25 的标准参数
- ✅ 避免参数冲突
- ✅ 更符合业界最佳实践

---

### 原因 3: 简化了代码逻辑 ⭐

**校准版本的复杂性**:
```python
# 需要计算平均长度
avg_length = sum(len(tokens) for tokens in corpus) / len(corpus)

# 需要调用校准函数
calibrated_score = self._calibrate_bm25_score(raw_score, len(chunk['content']), avg_length)

# 需要维护校准逻辑
def _calibrate_bm25_score(...):
    # 复杂的归一化逻辑
    ...
```

**原始分数版本的简洁性**:
```python
# 直接使用原始分数
refined_results.append({
    'score': float(raw_score),  # ✅ 简单直接
})

# 无需额外计算
# 无需维护校准逻辑
```

**优势**:
- ✅ 代码更简洁
- ✅ 更易维护
- ✅ 更少出错

   import { BM25Okapi } from 'rank-bm25';
   
   async function keywordSearch(query: string, topK: number) {
     // 1. FTS5 快速召回
     const candidates = await fts5Search(query, candidateSize);
     
     // 2. rank-bm25 重新计算（使用原始分数）
     const corpus = candidates.map(c => tokenize(c.content));
     const bm25 = new BM25Okapi(corpus);
     const scores = bm25.get_scores(tokenize(query));
     
     // 3. 按原始分数排序
     return candidates
       .map((c, i) => ({ ...c, score: scores[i] }))
       .sort((a, b) => b.score - a.score)

   }
