**A: 不是！候选集大小合理，rank-bm25 有真正的筛选作用。**

**证据**:
1. **候选集占比仅 25.6%**
   - 召回 50 个候选
   - 总分块 195 个
   - 占比: 50/195 = 25.6%

2. **提升幅度巨大**

   - 如果是"伪优化"，提升幅度应该很小

### 为什么大规模数据下 TypeScript 改进前表现下降？

**FTS5 bm25() 的问题在大规模数据下被放大**:

1. **IDF 计算不精确**
   - 小规模: 影响较小
   - 大规模: IDF 误差累积，导致分数失真

2. **长度归一化缺失**
   - 小规模: 文档长度差异不大
   - 大规模: 长短文档差异显著，缺少归一化导致偏差

3. **分数范围异常**
   - 小规模: 校准因子可能偶然有效
   - 大规模: 校准因子不适用，分数分布混乱

**rank-bm25 的优势**:
- ✅ 标准 IDF 计算，不受数据规模影响
- ✅ 完善的长度归一化
- ✅ 分数范围合理，无需校准

   ```typescript
   import { BM25Okapi } from 'rank-bm25';
   
   async function keywordSearch(query: string, topK: number) {
     // 1. FTS5 快速召回
     const candidates = await fts5Search(query, topK * 5);
     
     // 2. rank-bm25 重新计算
     const corpus = candidates.map(c => tokenize(c.content));
     const bm25 = new BM25Okapi(corpus);
     const scores = bm25.get_scores(tokenize(query));
     
     // 3. 按新分数排序
     return candidates
       .map((c, i) => ({ ...c, score: scores[i] }))
       .sort((a, b) => b.score - a.score)
       .slice(0, topK);
   }
   ```
