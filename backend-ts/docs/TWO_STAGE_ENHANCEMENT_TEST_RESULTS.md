## 💡 问题诊断

### 核心问题

1. **同义词扩展过于激进**
   ```python
   synonyms = {
       '知识': ['knowledge'],  # ❌ 太宽泛
       '数据库': ['database', 'db'],  # ✅ 合理
       '认证': ['auth', 'authentication', 'jwt'],  # ✅ 有效
   }
   ```
   
   **问题**: "知识" → "knowledge" 会匹配到所有包含 "knowledge" 的文档，包括不相关的

2. **短语匹配策略不够智能**
   - 当前：对所有中文短语都使用引号包裹
   - 问题：有些短语不应该被当作整体匹配

3. **长度归一化因子可能不合适**
   ```python
   length_factor = (avg_length / content_length) ** 0.5
   ```
   - 可能导致短文档得分虚高

### 改进建议

#### P0: 修复同义词扩展

**当前问题代码**:
```python
synonyms = {
    '知识': ['knowledge'],  # ❌ 太宽泛
    '向量': ['vector', 'embedding'],  # ⚠️ 可能过宽
}
```

**改进方案**:
```python
# 只保留高度相关的同义词
synonyms = {
    '迁移': ['migration'],  # 高度相关
    '数据库': ['database'],  # 高度相关
    '认证': ['auth', 'jwt'],  # 高度相关
    '优化': ['optimization'],  # 高度相关
    # 移除过于宽泛的同义词
    # '知识': ['knowledge'],  # ❌ 删除
    # '向量': ['embedding'],  # ❌ 删除
}
```

**预期效果**: 减少噪音，提升准确性

---

#### P1: 优化短语匹配策略

**当前问题**:
```python
phrases = [
    '向量搜索', '知识库', '数据库迁移', '混合搜索',
    '性能优化', 'API 认证', 'BM25 评分', '语义搜索',
    '关键词搜索', '全文检索', '倒排索引'
]
```

**改进方案**:
```python
# 只在查询中检测到这些短语时才使用
def detect_phrases(query: str) -> List[str]:
    detected = []
    for phrase in phrases:
        if phrase in query:
            detected.append(phrase)
    return detected

# 只对检测到的短语使用引号包裹
detected_phrases = detect_phrases(query)
for phrase in detected_phrases:
    processed_text = processed_text.replace(phrase, f'"{phrase}"')
```

**预期效果**: 避免过度使用短语匹配

---

#### P2: 调整长度归一化

**当前实现**:
```python
length_factor = (avg_length / content_length) ** 0.5
```

**改进方案**:
```python
# 使用更温和的归一化
length_factor = (avg_length / content_length) ** 0.3  # 降低指数

# 或者设置上下限
length_factor = max(0.5, min(2.0, (avg_length / content_length) ** 0.5))
```

**预期效果**: 避免极端情况下的分数失真
